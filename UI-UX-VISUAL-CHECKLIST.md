# UI/UX Visual Audit Checklist

Goal: Review the site's visual quality, usability, responsiveness, and conversion clarity one item at a time before making changes.

---

## 1. Preparation

- [ ] Confirm the local site runs correctly in the browser
- [ ] Confirm shared CSS and JS load on all key templates
- [ ] Identify the key pages for review:
  - [ ] Homepage
  - [ ] About page
  - [ ] Contact page
  - [ ] Services index
  - [ ] Main service pages
  - [ ] Main service area pages

---

## 2. Visual Hierarchy

- [ ] Hero section is clear within the first screen view
- [ ] Primary headline is easy to scan
- [ ] Supporting text is not too dense
- [ ] Main CTA is visually dominant
- [ ] Secondary CTA does not compete with the main CTA
- [ ] Important trust signals are visible early
- [ ] Section order feels logical from top to bottom

---

## 3. Layout and Spacing

- [ ] Content width is comfortable to read
- [ ] Section spacing is consistent
- [ ] Cards, panels, and boxes align cleanly
- [ ] Text blocks are not cramped
- [ ] No visually awkward gaps between sections
- [ ] No elements feel misaligned or off-balance
- [ ] Sidebar or support panels do not break the layout

---

## 4. Typography

- [ ] Font sizes are readable on desktop
- [ ] Font sizes are readable on mobile
- [ ] Headings have a clear scale hierarchy
- [ ] Paragraph line length is comfortable
- [ ] Paragraph spacing improves readability
- [ ] Bold text is used intentionally, not excessively
- [ ] Phone numbers and CTA text are easy to read quickly

---

## 5. Colors and Contrast

- [ ] Brand colors feel consistent across pages
- [ ] CTA buttons stand out clearly
- [ ] Text contrast is strong enough for easy reading
- [ ] Background sections do not reduce readability
- [ ] Accent colors are used consistently
- [ ] Urgent service messaging looks trustworthy, not aggressive

---

## 6. Navigation and Orientation

- [ ] Main navigation is clear and easy to use
- [ ] Current page context is obvious
- [ ] Important pages are reachable within a few clicks
- [ ] Internal links help users continue naturally
- [ ] Service pages link well to area pages
- [ ] Area pages link well to service pages
- [ ] Header and footer navigation feel complete

---

## 7. Mobile UX

- [ ] No horizontal scrolling on mobile
- [ ] Header works correctly on small screens
- [ ] Mobile menu is easy to open and close
- [ ] CTA remains visible without feeling intrusive
- [ ] Buttons and links are easy to tap
- [ ] Text remains readable without zooming
- [ ] Cards stack cleanly on narrow screens
- [ ] Images scale correctly on mobile

---

## 8. Images and Visual Assets

- [ ] All images load correctly
- [ ] No broken image placeholders appear
- [ ] Images are sharp enough for the layout size
- [ ] Images are cropped appropriately
- [ ] Images support trust and service credibility
- [ ] Decorative graphics do not distract from conversion
- [ ] Icons are consistent in style and size

---

## 9. Conversion UX

- [ ] Phone number is prominent in key areas
- [ ] Emergency intent is handled clearly
- [ ] Main CTA copy is specific and actionable
- [ ] Contact options are easy to find
- [ ] Trust signals support the call to action
- [ ] Service pages make the next step obvious
- [ ] Contact page reduces friction for urgent users
- [ ] Repeated CTAs are consistent across the site

---

## 10. Trust and Credibility

- [ ] The site looks like a real local service business
- [ ] The design supports reliability and urgency
- [ ] Testimonials or review elements are visually convincing
- [ ] Licensing, experience, or service claims feel credible
- [ ] Social links and business references appear intentional
- [ ] No section feels like placeholder content

---

## 11. Consistency Across Templates

- [ ] Homepage style matches inner pages
- [ ] Service pages follow a consistent structure
- [ ] Area pages follow a consistent structure
- [ ] Button styles are consistent everywhere
- [ ] Form styles are consistent everywhere
- [ ] Heading spacing is consistent everywhere
- [ ] CTA sections feel part of one system

---

## 12. Accessibility Items That Affect Visual UX

- [ ] Heading order supports visual structure
- [ ] Focus states are visible on interactive elements
- [ ] Text contrast is sufficient in all sections
- [ ] Link styling is distinguishable from plain text
- [ ] Images that matter have meaningful alt text
- [ ] Visual cues are not color-only

---

## 13. Performance Signals That Affect UX

- [ ] Hero content appears quickly
- [ ] Large images do not delay first impression
- [ ] Layout does not shift during load
- [ ] Buttons and menus respond quickly
- [ ] No obvious render-blocking visual delays

---

## 14. Review Output Format

- [ ] Mark each issue by severity: Critical / High / Medium / Low
- [ ] Note which page the issue appears on
- [ ] Note whether it is a visual issue, UX issue, or both
- [ ] Add fix recommendation for each issue
- [ ] Capture screenshots for issues that need visual proof

---

## 15. Execution Order

- [ ] Review homepage first
- [ ] Review mobile homepage second
- [ ] Review contact flow third
- [ ] Review service page templates fourth
- [ ] Review service area templates fifth
- [ ] Review consistency across all remaining pages last
